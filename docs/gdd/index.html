GDD: VibePins

Anti-Flatline guardrail. This GDD is a directory tree, not a single file. Each requirement is its own file. Coverage rows in docs/GDD_COVERAGE.json are written at requirement granularity, NOT chapter granularity. A multi-week project should produce on the order of 100+ rows, not 11. If you only have a dozen rows, your coverage is too coarse and the loop will self-terminate before the product is good.

Project pitch: A duckpin bowling game

How to use this directory

  • Each docs/gdd/<n>-<title>.html file is one requirement or one tightly-scoped section.
  • Each file starts with a <p data-status="..."> element: not_started | partial | done.
  • Once a file's work ships, append a <section data-role="build-log"> with what landed, the key files, and any non-obvious decisions. Build logs grow with the code.
  • Keep file names short and stable. The file path is referenced from GDD_COVERAGE.json.

Conventions

  • One requirement, one file, one row in the coverage ledger.
  • File names: <NN>-<kebab-title>.html, e.g. 05-vehicle-physics.html, 12-leaderboard-submit.html.
  • Cross-references between sections use relative links.
  • The em-dash ban applies here.

Index

Add entries below as sections are drafted. Each entry: filename + one-line description. This index is the human-readable map; the machine-readable map is docs/GDD_COVERAGE.json.

  • 01-vision-and-pillars.html: what VibePins is, the four design pillars, audience, and success criteria.
  • 02-core-loop.html: the duckpin core loop, three-ball frames, scoring (strike, spare, flat ten, open), tenth frame, scoreboard.
  • 03-string-pinsetter.html: the signature string-reset system: tethers, free fall, tension detection, the reset cycle, respot, tangles.
  • 04-look-and-feel.html: the mechanical / machine / Rube Goldberg aesthetic, materials, lighting, electromechanical scoreboard, audio, juice.
  • 05-async-multiplayer.html: the headline social feature: asynchronous, turn-based matches handed off by link across a day.
  • 06-reuse-and-tech.html: stack, the Rapier decision, the serverless backend, and what to reuse from the sibling project Hoops and VibeKit.
  • 07-leaderboard.html: the global and daily leaderboard, fed by completed solo and multiplayer games.
  • 08-controls.html: the SNES Super Bowling three-step control scheme (line up, stop spin meter, stop power meter).
  • 99-out-of-scope.html: the explicit v1 fence.

Out of scope

A dedicated docs/gdd/99-out-of-scope.html file is the explicit fence. List anything that has been considered and rejected for v1, with a one-line rationale. The loop must not scope-creep into items listed there without approval.
