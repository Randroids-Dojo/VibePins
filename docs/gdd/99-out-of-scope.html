Out of scope (v1 fence)

Status: not_started

The fence

These items have been considered and deliberately rejected for v1. The loop must not scope-creep into them without explicit approval. Each line carries a one-line rationale.

  • Real-time / synchronous multiplayer. v1 multiplayer is asynchronous and turn-based only (see async multiplayer). No live shared sessions, presence, spectating, or matchmaking.
  • Accounts, friend graphs, sign-in. Identity is a local per-device token plus a display name. The global leaderboard (section 07) and matches work without accounts.
  • Alternative / arcade game modes. Exactly one mode ships: a real ten-frame, three-ball duckpin round. No timed, endless, distance, streak, heat, or bonus-scoring modes.
  • In-match chat. Players talk in whatever app they shared the handoff link through.
  • Tenpin, candlepin, or other bowling variants. VibePins is duckpin only. Variants dilute the focus.
  • Full explorable venue / walking around the bar. The scene is one lane in a suggested room, not a 3D environment to roam.
  • Other Pins Mechanical games (pinball, ping pong, cage soccer). Flavor references only; not playable.
  • Manual string-untangle minigame. Tangles resolve at the next re-rack. A manual untangle interaction is a later idea, not v1. See string pinsetter, tangles.
  • Pinsetter maintenance simulation (cord wear, motor faults). Real but not fun-bearing for v1.
  • Character / avatar customization, cosmetics, progression economy. No meta-progression in v1.
  • Native desktop / mobile app builds. Web (Vercel static) is the only v1 target.
