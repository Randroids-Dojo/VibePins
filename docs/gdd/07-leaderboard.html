Global leaderboard

Status: partial

Intent

A global leaderboard gives the single-player round a reason to come back: post your best honest ten-frame score and see where it lands. It reuses the proven serverless + Upstash Redis pattern from the sibling project Hoops (see reuse and tech), so leaderboard and async multiplayer share one backend approach.

What posts a score

  • A completed solo ten-frame game posts its authentic final score.
  • Each player's completed line in an async match is eligible to post the same way, so multiplayer also feeds the global board.
  • Only completed games count. An abandoned game posts nothing.
  • Scores are submitted with a display name (short, length-capped) and validated server-side against the duckpin scoring rules so an impossible score is rejected.

Boards

  • An all-time global top list (capped, for example top 100).
  • A daily board that resets each day, mirroring the Hoops pattern, so newcomers can chart without beating the all-time greats.
  • The player's own best is highlighted in context (their rank and nearby entries) even when off the visible top slice.

Display

The leaderboard is reachable from the main menu and shown on the end-of-game summary so a fresh score lands in context. Visual treatment follows the electromechanical scoreboard language from look and feel rather than a generic table: think a rolling-counter standings board.

What this does not do (v1)

  • No accounts or cross-device identity beyond the local player token; your "best" is per-device unless you reuse the same device.
  • No friend-only or private boards in v1 (global and daily only).
  • No seasons, prizes, or anti-cheat beyond server-side score validation.
