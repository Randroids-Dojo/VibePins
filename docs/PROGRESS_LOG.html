String Pinsetter Tether: Slack Overhead Cord Per Pin

Branch
feature/string-tether
PR
#TBD
Changed
Tethered each of the ten pins to the overhead drive point, the signature string-pinsetter mechanic. In src/pins.ts, every pin now also gets a per-pin fixed, collider-less anchor body directly above its home spot and a Rapier rope joint (a max-distance constraint, slack until taut) attaching the pin neck (a local point above the dropped belly/COM) to that anchor; a thin grey THREE.Line cord per pin tracks the neck each frame in PinSet.sync(). New TETHER config object in src/config.ts holds the tunables: anchor height 2.5m, neck offset 0.30*pinHeight, slack length 2.96m, cord colour. Slack length is derived from the worst-case anchor-to-neck distance (an outer back-row pin lying flat slid to the far deck corner, ~2.81m) plus a 0.15m margin, so the cord never restrains a natural fall (REQ-014) yet stays finite for a future reset lift (REQ-015). Exported pure helpers neckLocalAnchor() and tetherAnchorPositions().
Verification
em-dash grep clean; git diff --check clean; npm run typecheck clean; npm run test 23/23 passing; npm run build succeeds (known single-chunk size warning, F-003). Tests include a pure slack-length guard that recomputes the worst-case deck-fall distance, and a headless Rapier smoke proving (a) a struck pin's tethered fall matches its untethered fall to within 1cm with the cord never going taut, and (b) a freely dropped pin is caught at the slack length. A four-lens adversarial review workflow (correctness, GDD faithfulness, test rigor, style) confirmed the implementation; its two confirmed test-rigor findings (a missing motion lower-bound and a too-loose comparison tolerance) were fixed before this entry.
Assumptions
Anchor height 2.5m is the midpoint of the GDD 2.2-2.8m range; the neck offset 0.30*pinHeight and slack length 2.96m are first-pass tunables to tighten against the playtest gate (Q-010). REQ-014 free-fall is proven via a velocity-proxy strike; real-ball-impact validation awaits the ball slice, so REQ-014 stays partial.
GDD coverage
REQ-013 and REQ-015 flipped to done; REQ-014 to partial. docs/gdd/03-string-pinsetter.html status to partial with a build log entry.
Followups
None new. Detection (REQ-016/017) and the reset cycle (REQ-018/019) remain the next string-pinsetter slices; the anchor bodies are stored so the reset slice can reel them in.
